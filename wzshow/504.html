Linux统下如何为SFTP服务添加新用户

在Linux操作系统中，SFTP（安全文件传输协议）是一种通过SSH（安全外壳协议）进行文件传输的安全通信协议，它为文件传输提供了加密的安全环境，为了通过SFTP进行文件传输，通常需要在Linux系统中创建一个专用的SFTP用户，以下是在Linux系统中为SFTP服务添加新用户的详细步骤。

1. 确认SFTP服务已安装

在添加SFTP用户之前，首先需要确认SFTP服务是否已经安装，大多数Linux发行版默认安装了SSH服务，而SFTP服务通常与SSH服务一同安装，可以通过以下命令检查SSH服务是否已安装：

ssh番茄游戏网 -V

如果SSH服务未安装，可以使用以下命令进行安装：

基于Debian的系统（如Ubuntu）：

sudo apt-get update
sudo apt-get install openssh-server

基于Red Hat的系统（如CentOS）：

sudo yum install openssh-server

2. 新建SFTP用户账户

创建SFTP用户与创建普通Linux用户的过程相似，可以使用useradd命令创建用户：

sudo useradd -m -d /home/sftpuser -s /bin/bash sftpuser

-m参数用于创建用户的主目录，-d参数指定用户的主目录路径，-s参数指定用户的默认shell。

3. 设置用户密码

为SFTP用户设置密码，可以使用passwd命令：

布丁影院
sudo passwd sftpuser

按照提示输入两次密码即可。

4. 配置SFTP权限

为确保SFTP用户只能访问其家目录，需要修改用户的权限设置：

- 切换到root用户：

sudo su

- 将SFTP用户的家目录的所星辰影院有者改为用户本人：

chown sftpuser:sftpuser /home/sftpuser

- 将SFTP用户的家目录的权限设置为700，确保只有用户本人可以访问：

chmod 700 /home/sftpuser

- 允许用户通过SSH访问SFTP服务，编辑/etc/ssh/sshd_config文件：

sudo nano /etc/ssh/sshd_config

找到并修改以下行：

Subsystem sftp /usr/lib/openssh/sftp-server

确保Subsystem行后面没有注释符号（#）。

- 保存并退出文件。

- 重启SSH服务以应用更改：

sudo systemctl restart ssh

5. 创建SFTP用户的配置文件

在用户的主目录下创建一个名为.sftp的隐藏目录，并设置相应的权限：

sudo mkdir /home/sftpuser/.sftp
sudo chown sftpuser:sftpuser /home/sftpuser/.sftp
sudo chmod 700 /home/sftpuser/.sftp

在.sftp目录下创建一个名为授权文件名的文件，例如authorized_keys，用于存储允许访问SFTP的用户公钥：

sudo nano /home/sftpuser/.sftp/authorized_keys

将用户的公钥粘贴到该文件中，每行一个。

6. 进行SFTP连接

使用SFTP客户端连接到服务器，并尝试上传或下载文件，以下是使用lftp命令行工具进行SFTP连接的示例：

lftp -u sftpuser,sftpuserpass localhost

sftpuser是用户名，sftpuserpass是用户密码。

通过以上步骤，您已经成功在Linux系统中为SFTP服务添加了一个新用户，用户现在可以通过SFTP客户端安全地访问和传输文件。
